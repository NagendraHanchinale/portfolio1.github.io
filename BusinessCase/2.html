Surge Calculation

Problem

Uber’s surge pricing faces issues like user frustration due to high and unpredictable fares, especially during emergencies. Drivers may try to game the system to trigger higher prices. The pricing can vary sharply across nearby areas, leading to unfairness. Sometimes, the system sets incorrect surges due to poor demand prediction or lack of data.

1. Real‑Time Supply‑Demand Ratio Monitoring with H3

The city map is divided into uniform hexagonal H3 cells. Every few seconds, Uber counts active ride requests and available drivers inside each cell, computes the ratio [riders/max(drivers,1)], and feeds it through a smooth multiplier function. Because H3 cells share equal‑area neighbors, this avoids sharp price jumps across streets and updates fares precisely where demand outpaces supply.

Fig.1. The maps above depict the process of bucketing points with H3: cars in a city; cars in hexagons; and hexagons shaded by number of cars.

Explanation : Continuously monitors rider-driver ratios in geographic zones. When demand exceeds supply beyond a threshold, applies surge multiplier using economic equilibrium principles.

Fig. 2: Flowchart showing how surge pricing is calculated based on current demand and supply.

🔧 Code (C++)

#include <bits/stdc++.h>

class SurgePricingEngine {
private:
    std::unordered_map zoneMultipliers;  // Zone ID -> multiplier
    const double BASE_FARE = 5.0;                             // Base fare in dollars
    const double THRESHOLD_RATIO = 1.5;                       // Demand/supply ratio threshold

public:
    // Update pricing based on real-time ratios
    void updatePricing(const std::string& zoneId, double demand, double supply) {
        double ratio = demand / (supply + 1e-5);  // Avoid division by zero
        
        if (ratio > THRESHOLD_RATIO) {
            // Quadratic growth beyond threshold (more aggressive surge)
            double surge = 1.0 + 0.5 * pow(ratio - THRESHOLD_RATIO, 1.5);
            zoneMultipliers[zoneId] = std::min(surge, 5.0);  // Cap at 5x
        } else {
            zoneMultipliers[zoneId] = 1.0;  // No surge
        }
    }

    // Calculate final fare
    double calculateFare(const std::string& zoneId, double distance, double duration) {
        double baseFare = BASE_FARE + 1.5 * distance + 0.3 * duration;
        return baseFare * zoneMultipliers[zoneId];
    }
};

3: Hybrid Real-Time + Predictive Model

Explanation : Uses historical data to predict demand spikes. Combines time patterns, weather, and events to preemptively adjust pricing.

🔧 Code (C++)

#include <bits/stdc++.h>

class HybridPricingModel {
private:
    double alpha = 0.7;  // Weight for predictive model
    double beta = 0.3;   // Weight for real-time data

public:
    double calculateSurgeMultiplier(
        double predictedDemand, 
        double realTimeDemand,
        double driverSupply
    ) {
        // Weighted demand estimate
        double hybridDemand = alpha * predictedDemand + beta * realTimeDemand;
        double ratio = hybridDemand / (driverSupply + 1e-5);
        
        // Sigmoid surge function
        double surge = 1.0 + 4.0 / (1.0 + exp(-0.5*(ratio - 2.0)));
        return std::clamp(surge, 1.0, 5.0);
    }
};

// Usage in pricing system
void updateZonePricing(const std::string& zoneId) {
    double predDemand = forecastModel.predict(zoneId);
    double realDemand = getActiveRiders(zoneId);
    double supply = getAvailableDrivers(zoneId);
    
    double multiplier = hybridModel.calculateSurgeMultiplier(
        predDemand, realDemand, supply
    );
    
    setZoneMultiplier(zoneId, multiplier);
}
